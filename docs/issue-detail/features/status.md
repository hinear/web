# Status Feature Spec

## 목적

사용자가 이슈의 현재 진행 상태를 관리하고, `Triage`에서 실행 상태로 넘길 수 있게 한다.

## 상태 집합

- `Triage`
- `Backlog`
- `Todo`
- `In Progress`
- `Done`

## 사용자 스토리

- 사용자는 신규 이슈가 `Triage` 상태로 시작하는 것을 확인할 수 있다.
- 사용자는 상태 셀렉터에서 다른 상태를 선택할 수 있다.

## 요구사항

- 신규 이슈 기본 상태는 `Triage`다.
- 상태 셀렉터는 허용 상태만 표시한다.
- `Triage`에서 `Backlog` 또는 `Todo`로 이동할 수 있다.
- 상태 변경 성공 시 activity log에 남는다.
- 상태 변경 실패 시 마지막 저장 상태로 롤백한다.

## UX 원칙

- 상태 변경은 빠르게 접근 가능해야 한다.
- 현재 상태는 시각적으로 분명해야 한다.

## 테스트 포인트

- 신규 이슈 기본 상태가 `Triage`다
- 상태 셀렉터에 모든 허용 상태가 보인다
- `Triage -> Backlog` 변경 가능
- `Triage -> Todo` 변경 가능
- 실패 시 이전 상태로 롤백
- 상태 변경 activity log가 표시된다
