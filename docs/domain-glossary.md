# Hinear 도메인 용어사전

Hinear 프로젝트에서 사용하는 도메인 용어들의 정의입니다.

## 프로젝트 (Project)

**정의**: 이슈들을 관리하는 최상위 작업 공간

**특징**:
- `personal` (개인) 또는 `team` (팀) 타입으로 구분
- 각 프로젝트는 고유한 키(key)를 가짐 (예: `WEB`, `MOBILE`)
- 프로젝트 식별자는 자동 생성됨 (UUID)

**예시**:
- "WEB 프로젝트에 새 이슈를 생성하세요"
- "이 프로젝트는 팀 프로젝트입니다"

**관련 용어**: 워크스페이스, Issue Board

---

## 이슈 (Issue)

**정의**: 프로젝트 내에서 해결해야 할 작업 단위

**특징**:
- 프로젝트에 속해 있음
- 고유한 식별자를 가짐 (예: `WEB-1`, `SAS-42`)
- 상태(status), 우선순위(priority), 담당자(assignee)를 가짐
- 기본 상태는 `Triage`

**식별자 형식**: `[PROJECT_KEY]-[숫자]`
- `WEB-1` (WEB 프로젝트의 1번 이슈)
- `SAS-42` (SAS 프로젝트의 42번 이슈)

**예시**:
- "WEB-1 이슈를 할당하세요"
- "이 이슈의 상태를 In Progress로 변경하세요"

**관련 용어**: 이슈 번호, 이슈 식별자

---

## 이슈 번호 (Issue Number)

**정의**: 프로젝트 내에서 이슈를 식별하는 숫자

**특징**:
- 프로젝트별로 1부터 시작하는 순차적 번호
- 프로젝트 삭제 시 재사용되지 않음
- 이슈 식별자의 구성 요소

**예시**:
- "이슈 번호 42번"
- "WEB-42에서 42가 이슈 번호"

**관련 용어**: 이슈 식별자, 프로젝트 키

---

## 이슈 식별자 (Issue Identifier)

**정의**: 프로젝트 키와 이슈 번호를 조합한 고유 식별자

**형식**: `[PROJECT_KEY]-[ISSUE_NUMBER]`
- 대문자 알파벳 + 하이픈 + 숫자
- 예: `WEB-1`, `SAS-42`, `MOBILE-100`

**특징**:
- 프로젝트 간에도 고유함
- 사용자에게 친숙한 형태
- URL 경로, 참조, UI 표시에 사용

**예시**:
- "WEB-1 이슈를 참조하세요"
- "이슈 식별자를 복사해서 이슈를 언급하세요"

**관련 용어**: 프로젝트 키, 이슈 번호

---

## 프로젝트 키 (Project Key)

**정의**: 프로젝트를 식별하는 짧은 코드

**특징**:
- 대문자 알파벳과 숫자로 구성 (예: `WEB`, `MOBILE`, `API`)
- 최소 2자, 최대 10자
- 영문 대문자로 시작
- 프로젝트 생성 시 사용자가 지정
- 이슈 식별자의 접두어로 사용

**규칙**:
- 영문 대문자로 시작
- 영문 대문자와 숫자만 사용 가능
- 중복 불가

**예시**:
- "WEB 프로젝트의 키는 WEB입니다"
- "MOBILE-1 이슈에서 MOBILE가 프로젝트 키"

**관련 용어**: 이슈 식별자

---

## Issue Board

**정의**: 이슈들을 상태별로 시각화하여 관리하는 화면

**특징**:
- 칸반(Kanban) 스타일로 이슈를 표시
- 상태별 컬럼으로 구분 (Triage, Backlog, Todo, In Progress, Done 등)
- 드래그앤드롭으로 상태 변경 가능
- 프로젝트의 메인 화면

**상태 컬럼**:
- **Triage**: 초기 상태, 분류 필요
- **Backlog**: 진행 예정인 작업
- **Todo**: 할 일 목록
- **In Progress**: 진행 중인 작업
- **Done**: 완료된 작업
- **Canceled**: 취소된 작업

**예시**:
- "이슈보드에서 이슈를 드래그해서 상태를 변경하세요"
- "이슈보드를 새로고침하세요"

**관련 용어**: 칸반 보드, 워크스페이스

---

## Overview

**정의**: 프로젝트의 전체 현황과 통계를 한눈에 보여주는 화면

**특징**:
- 프로젝트 요약 정보 제공
- 이슈 통계 (상태별, 우선순위별 등)
- 최근 활동 내역
- 진행 상황 시각화

**용도**:
- 프로젝트 현황 빠른 파악
- 팀 성과 모니터링
- 병목 지점 식별
- 전체 흐름 이해

**예시**:
- "오버뷰에서 프로젝트 현황을 확인하세요"
- "오버뷰를 통해 통계를 보세요"

**관련 용어**: 이슈보드, 대시보드

---

## 워크스페이스 (Workspace)

**정의**: 프로젝트의 작업 공간을 보여주는 화면

**특징**:
- 프로젝트 관련 모든 기능에 접근
- Issue Board, Overview, Settings 등으로 네비게이션
- 사이드바, 메인 컨텐츠 영역으로 구성

**구성 요소**:
- 사이드바 (내비게이션)
- Issue Board 영역
- 프로젝트 정보

**예시**:
- "워크스페이스로 돌아가기"
- "워크스페이스에서 프로젝트를 전환하세요"

**관련 용어**: 프로젝트, Issue Board

---

## 드로어 (Drawer)

**정의**: 화면 가장자리에서 슬라이드되어 나타나는 패널

**특징**:
- 오른쪽에서 슬라이드되어 나타남
- 페이지 이동 없이 이슈 상세를 볼 수 있음
- URL 파라미터로 제어 (`?drawer=WEB-1`)
- 백드롭 클릭 또는 ESC 키로 닫기 가능

**용도**:
- 이슈 빠른 확인
- Issue Board에서 벗어나지 않고 이슈 편집
- 전체 페이지로 전환 가능

**URL 패턴**:
- 드로어: `/projects/[projectId]?drawer=[issueIdentifier]`
- 전체 페이지: `/projects/[projectId]/issues/[issueId]?view=full`

**예시**:
- "드로어에서 이슈를 빠르게 확인하세요"
- "드로어를 닫고 이슈보드로 돌아가세요"

**관련 용어**: 전체 페이지, 모달

---

## 상태 (Status)

**정의**: 이슈의 현재 진행 상태를 나타내는 분류

**상태 목록**:
1. **Triage** (트리아지): 초기 상태, 분류 필요
2. **Backlog** (백로그): 진행 예정인 작업
3. **Todo** (할 일): 해야 할 작업
4. **In Progress** (진행 중): 현재 진행 중
5. **Done** (완료): 완료된 작업
6. **Canceled** (취소): 취소된 작업

**특징**:
- Issue Board에서 컬럼으로 표시
- 드래그앤드롭으로 변경 가능
- 각 상태에 따른 색상 구분

**예시**:
- "이슈 상태를 In Progress로 변경하세요"
- "Triage 상태의 이슈들을 분류하세요"

**관련 용어**: Issue Board, 칸반

---

## 우선순위 (Priority)

**정의**: 이슈의 중요도나 긴급성을 나타내는 등급

**우선순위 목록**:
1. **Urgent** (긴급): 즉시 처리 필요
2. **High** (높음): 높은 우선순위
3. **Medium** (중간): 일반적인 우선순위
4. **Low** (낮음): 낮은 우선순위
5. **No Priority** (우선순위 없음): 미지정

**특징**:
- 이슈 카드에 색상으로 표시
- 정렬 기준으로 활용 가능

**예시**:
- "긴급 이슈를 먼저 처리하세요"
- "우선순위를 높게 설정하세요"

**관련 용어**: 이슈, Issue Board

---

## 담당자 (Assignee)

**정의**: 이슈를 담당하여 처리하는 사람

**특징**:
- 프로젝트 멤버 중에서 지정
- 여러 명을 지정할 수 없음 (1인당 1명)
- 미지정일 수 있음 (Unassigned)

**예시**:
- "이슈에 담당자를 배정하세요"
- "담당자가 없는 이슈들을 확인하세요"

**관련 용어**: 프로젝트 멤버, 멤버

---

## 프로젝트 멤버 (Project Member)

**정의**: 프로젝트에 참여하고 있는 사람

**역할**:
- **Owner** (소유자): 프로젝트 관리자, 모든 권한 보유
- **Member** (멤버): 일반 참여자, 제한된 권한

**특징**:
- 프로젝트별로 멤버 관리
- 초대(invitation)로 추가 가능
- 역할에 따라 권한 차이

**예시**:
- "프로젝트에 새 멤버를 초대하세요"
- "프로젝트 소유자만 설정을 변경할 수 있습니다"

**관련 용어**: 담당자, 초대

---

## 프로젝트 소유자 (Project Owner)

**정의**: 프로젝트를 생성하고 관리하는 사람

**권한**:
- 프로젝트 설정 변경
- 멤버 초대 및 역할 관리
- 이슈 CRUD 모든 권한
- 프로젝트 삭제 권한

**특징**:
- 프로젝트 생성 시 자동으로 소유자 지정
- 프로젝트당 1명 이상의 소유자 가능
- 소유자 권한을 다른 멤버에게 위임 불가능

**예시**:
- "프로젝트 소유자만 프로젝트를 삭제할 수 있습니다"
- "프로젝트 소유자가 설정을 관리하세요"

**관련 용어**: 프로젝트 멤버, 멤버

---

## 라벨 (Label)

**정의**: 이슈를 분류하고 시각적으로 구분하기 위한 태그

**특징**:
- 프로젝트별로 생성
- 색상 자동 지정
- 하나의 이슈에 여러 라벨 부착 가능
- 중복된 이름의 라벨은 자동으로 병합

**용도**:
- 이슈 카테고리 분류
- 작업 유형 구분
- 팀/모듈 식별
- 우선순위 시각화

**예시**:
- "이슈에 'bug' 라벨을 추가하세요"
- "라벨로 이슈를 필터링하세요"

**관련 용어**: 이슈, 태그

---

## 활동 로그 (Activity Log)

**정의**: 이슈의 변경 이력을 기록한 로그

**기록 정보**:
- 변경한 사람 (actor)
- 변경 시간 (timestamp)
- 변경 유형 (type)
- 변경 필드 (field)
- 이전 값 (from)
- 새로운 값 (to)
- 변경 요약 (summary)

**변경 유형 예시**:
- `issue.title.updated`: 제목 변경
- `issue.status.updated`: 상태 변경
- `issue.assignee.updated`: 담당자 변경
- `issue.priority.updated`: 우선순위 변경

**예시**:
- "활동 로그에서 변경 이력을 확인하세요"
- "누가 이슈를 변경했는지 활동 로그를 보세요"

**관련 용어**: 이슈, 변경 이력

---

## 댓글 (Comment)

**정의**: 이슈에 대한 토론, 질문, 피드백을 남기는 메시지

**특징**:
- 이슈에 작성
- 작성자 정보와 작성 시간 포함
- 마크다운 형식 지원 (예정)
- 활동 로그에 기록됨

**용도**:
- 이슈에 대한 논의
- 질문과 답변
- 피드백 및 리뷰
- 의사소통 기록

**예시**:
- "이슈에 댓글을 남기세요"
- "최근 댓글을 확인하세요"

**관련 용어**: 이슈, 활동 로그

---

## 프로젝트 식별자 (Project ID)

**정의**: 시스템 내에서 프로젝트를 식별하는 고유 ID

**특징**:
- UUID 형식 (예: `0e91d9f3-5791-43e6-9ea6-f503478a506d`)
- 사용자가 직접 보거나 사용할 일 없음
- 주로 URL 경로에서 사용

**예시**:
- "프로젝트 ID로 데이터를 조회하세요"
- "URL의 프로젝트 ID로 프로젝트를 식별합니다"

**관련 용어**: 프로젝트 키, 이슈 ID

---

## 이슈 ID (Issue ID)

**정의**: 시스템 내에서 이슈를 식별하는 고유 ID

**특징**:
- UUID 형식
- 사용자가 직접 보거나 사용할 일 없음
- 데이터베이스 조회, API 호출에 사용

**예시**:
- "이슈 ID로 이슈를 조회하세요"
- "이슈 ID로 댓글을 연결하세요"

**관련 용어**: 이슈 식별자, 프로젝트 ID

---

## 버전 (Version)

**정의**: 이슈의 동시 수정 충돌을 방지하기 위한 낙관적 락킹 버전

**특징**:
- 이슈 수정 시마다 1씩 증가
- 업데이트 시 현재 버전을 함께 전송
- 버전 불일치 시 충돌 에러 발생

**충돌 시나리오**:
1. 사용자 A가 이슈 로드 (버전 1)
2. 사용자 B가 같은 이슈 수정 (버전 2로 저장)
3. 사용자 A가 수정 시도 (버전 1로 전송)
4. 버전 불일치로 충돌 발생
5. 최신 데이터를 다시 로드해서 충돌 해결

**예시**:
- "이슈 버전이 변경되었습니다. 다시 로드하세요"
- "충돌이 발생했습니다. 최신 내용을 확인하세요"

**관련 용어**: 낙관적 락킹, 충돌, 동시성

---

## 낙관적 락 (Optimistic Lock)

**정의**: 데이터 충돌을 방지하기 위한 동시성 제어 메커니즘

**원리**:
- 버전 번호를 사용하여 데이터 변경 감지
- 수정 시 버전 확인 후 업데이트
- 충돌 시 사용자에게 알리고 해결 요청

**장점**:
- 데이터 무결점 보장
- 동시 수정 충돌 방지
- 사용자에게 명확한 피드백 제공

**예시**:
- "낙관적 락을 사용해서 동시 수정을 방지하세요"
- "버전 충돌이 발생했습니다"

**관련 용어**: 버전, 충돌

---

## 전체 페이지 (Full Page)

**정의**: 이슈 상세를 전체 화면으로 보는 모드

**특징**:
- 별도의 URL 경로 (`/projects/[projectId]/issues/[issueId]`)
- `?view=full` 파라미터로 명시
- 모든 이슈 정보와 기능에 접근
- 뒤로가기/앞으로가기 정상 작동

**용도**:
- 이슈 상세 전체 편집
- 복잡한 작업 수행
- 집중 필요한 작업

**URL 패턴**:
- 전체 페이지: `/projects/[projectId]/issues/[issueId]?view=full`
- 드로어: `/projects/[projectId]?drawer=[issueIdentifier]`

**예시**:
- "전체 페이지에서 이슈를 편집하세요"
- "전체 페이지로 이동하세요"

**관련 용어**: 드로어, 모달

---

## 초대 (Invitation)

**정의**: 프로젝트에 새 멤버를 추가하기 위한 초대장

**상태**:
- **Pending** (대기): 수락 대기 중
- **Accepted** (수락): 초대 수락 완료
- **Revoked** (철회): 초대 철회됨
- **Expired** (만료): 초대 만료됨

**특징**:
- 이메일로 초대장 발송
- 토큰 기반 수락 링크
- 만료 기간 설정 가능
- 소유자만 초대 가능

**예시**:
- "팀원을 프로젝트에 초대하세요"
- "초대가 대기 중입니다"

**관련 용어**: 프로젝트 멤버, 소유자

## 개인 프로젝트 (Personal Project)

**정의**: 한 사람이 관리하는 개인용 프로젝트

**특징**:
- 소유자 1명만 가능
- 멤버 초대 불가
- 개인 작업 관리에 최적화

**예시**:
- "개인 프로젝트를 생성하세요"
- "개인용으로 사용하는 프로젝트입니다"

**관련 용어**: 팀 프로젝트, 프로젝트

---

## 팀 프로젝트 (Team Project)

**정의**: 여러 사람이 함께 협업하는 팀용 프로젝트

**특징**:
- 소유자 1명 이상
- 멤버 초대 가능
- 역할 기반 권한 관리
- 협업 기능 활용

**예시**:
- "팀 프로젝트를 생성하고 팀원을 초대하세요"
- "팀원들과 함께 이슈를 관리하세요"

**관련 용어**: 개인 프로젝트, 프로젝트 멤버

---

## 심각한 (Critical)

**정의**: 사용할 수 없는 상태

**사용 예시**:
- 이슈 데이터를 찾을 수 없음
- 프로젝트에 접근 권한 없음
- 필수 데이터 누락

**사용 방법**:
- 주요 기능 장애 시
- 데이터 무결점 위반 시
- 보안 문제 발생 시

## 경고 (Warning)

**정의**: 주의가 필요한 상태

**사용 예시**:
- 기능 제한이 있음
- 데이터 부분 누락
- 권한 부족

**사용 방법**:
- 기능 제한 안내
- 데이터 확인 필요
- 권한 요구
